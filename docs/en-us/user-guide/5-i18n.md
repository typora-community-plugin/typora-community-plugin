# Internationalization (i18n)

Typora Community Plugin provides language switching functionality, allowing users to freely choose the display language of the plugin.


## Switching Languages

In Typora's settings interface, go to the **About** tab and find the **Language** option:

- Click the dropdown menu and select your desired language
- After selection, the plugin will automatically save the setting and take effect

> Note: You need to **restart Typora** for language changes to fully take effect.


## Supported Languages

Currently supported languages include:

| Code | Name |
|------|------|
| `en` | English |
| `zh-cn` | Simplified Chinese |
| `de` | Deutsch |

More languages can be added by submitting a Pull Request.


## Internationalization File Structure

The plugin's internationalization resources are stored in the `packages/core/src/locales/` directory, using JSON format:

```
src/locales/
├── lang.en.json
├── lang.zh-cn.json
└── lang.de.json
```

Each language file has consistent key structures, corresponding to UI text for each module:

```json
{
  "settingTabs": {
    "about": {
      "name": "About",
      "projectDesc": "A community plugin system for",
      "labelVersion": "Version",
      "buttonUpdate": "Update",
      "lang": "Language"
    }
  }
}
```

## Contributing Translations

If you would like to add a new language translation for the plugin:

1. Create a new language file, such as `lang.ja.json`
2. Refer to existing language files and translate all values into the target language
3. Submit a Pull Request

No code logic modifications are needed; simply add the corresponding JSON translation file.
